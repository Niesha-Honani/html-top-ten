1. Grid CSS styling

The CSS grid layout module excels at dividing a page into major regions or defining the relationship in terms of size, position, and layering between parts of a control built from HTML primitives.
Like tables, grid layout enables an author to align elements into columns and rows. However, many more layouts are either possible or easier with CSS grid than they were with tables. For example, a grid container's child elements could position themselves so they actually overlap and layer, similar to CSS positioned elements.

Example of Grid Layout

          
          .grid-container{
              display: grid;
              grid-template-areas: 3;
              grid-template-columns: repeat(2, 1fr);
              gap: 10px;
              grid-auto-rows: 100px, auto;
          }
          .grid-header{
              grid-column:1/span 2;
              padding:20px;
              text-align: center;
              font-size: 1.5em;
              color:#57f2ff;
              background-color: #773399;
              border: double 15px #57f2ff;
          }

          .grid-item{
              background-color: lightblue;
              padding: 1em;
              border: 1px solid blue;
          }

          .grid-column-left{
              background-color:lightgreen;
              padding: 20px;
          }

2. CSS Functions | circle() - clip-path

The circle() CSS function defines a circle using a radius and a position. It is one of the data types.

CSS Functions are pretty neat because it takes in parameters so that you can do some pretty interesting stuff besides change the image shape. Another one, is calc() and blur().

          
            .skeletor-image{
              display:block;
              width: 15%;
              height: auto;
              margin: auto;
             
              shape-outside: circle(50%);
              clip-path: ellipse(75% 100% at 50% 100%);
              
              border: solid 10px black;
          }

I used this snippet to create the ellipse around the Skeletor image.
The one drawback I didn't compensate for is viewport sizing, that can affect how the styling works in the function. I originally tried an arch, but I didn't adjust so the function would resize to the viewport sizes. It's a little more in-depth.

3. CSS Font Styling | Everything

CSS Font styling has so many different ways to style font, it's hard to choose just one. So I made a full styling of the main H1 "Top Ten" I'm not a styling genious with fonts, I pretty much looked at some website fonts that are trending and copied some of the styling values and played with the values using inspector.

Interesting Font Styling

  • text-transform: sets everything to uppercase
  • letter-spacing: adds spacing in between letters
  • transform: translate(-50%, -50%) - The transform CSS property lets you rotate, scale, skew, or translate an element. It modifies the coordinate space of the CSS visual formatting model.
  • z-index: The z-index CSS property sets the z-order of a positioned element and its descendants or flex and grid items. Overlapping elements with a larger z-index cover those with a smaller one.
  • opacity: Makes whatever element or font transparent.
  • pointer-events: The pointer-events CSS property sets under what circumstances (if any) a particular graphic element can become the target of pointer events. Side Note: This is also good if your Font or element overlaps into a clickable area 
            
              .grid-header h1{
                font-size: clamp(3rem, 8vw, 7rem);
                text-transform: uppercase;
                letter-spacing: 0.08em;
                color: yellow;
            
                position: absolute;
                left:55%;
                top: 45%;
                transform: translate(-50%, -50%);
            
                text-shadow:
                    1px 2px 2px #ff66e9,
                    0 .3em .7em black;
                opacity: .80;
                z-index: 1;
                pointer-events: none;
            }
