1. Grid CSS styling

The CSS grid layout module excels at dividing a page into major regions or defining the relationship in terms of size, position, and layering between parts of a control built from HTML primitives.
Like tables, grid layout enables an author to align elements into columns and rows. However, many more layouts are either possible or easier with CSS grid than they were with tables. For example, a grid container's child elements could position themselves so they actually overlap and layer, similar to CSS positioned elements.

Example of Grid Layout

          
          .grid-container{
              display: grid;
              grid-template-areas: 3;
              grid-template-columns: repeat(2, 1fr);
              gap: 10px;
              grid-auto-rows: 100px, auto;
          }
          .grid-header{
              grid-column:1/span 2;
              padding:20px;
              text-align: center;
              font-size: 1.5em;
              color:#57f2ff;
              background-color: #773399;
              border: double 15px #57f2ff;
          }

          .grid-item{
              background-color: lightblue;
              padding: 1em;
              border: 1px solid blue;
          }

          .grid-column-left{
              background-color:lightgreen;
              padding: 20px;
          }

2. CSS Functions | circle() - clip-path

The circle() CSS function defines a circle using a radius and a position. It is one of the data types.

CSS Functions are pretty neat because it takes in parameters so that you can do some pretty interesting stuff besides change the image shape. Another one, is calc() and blur().

          
            .skeletor-image{
              display:block;
              width: 15%;
              height: auto;
              margin: auto;
             
              shape-outside: circle(50%);
              clip-path: ellipse(75% 100% at 50% 100%);
              
              border: solid 10px black;
          }

I used this snippet to create the ellipse around the Skeletor image.
The one drawback I didn't compensate for is viewport sizing, that can affect how the styling works in the function. I originally tried an arch, but I didn't adjust so the function would resize to the viewport sizes. It's a little more in-depth.

3. CSS Font Styling | Everything

CSS Font styling has so many different ways to style font, it's hard to choose just one. So I made a full styling of the main H1 "Top Ten" I'm not a styling genious with fonts, I pretty much looked at some website fonts that are trending and copied some of the styling values and played with the values using inspector.

Interesting Font Styling

  • text-transform: sets everything to uppercase
  • letter-spacing: adds spacing in between letters
  • transform: translate(-50%, -50%) - The transform CSS property lets you rotate, scale, skew, or translate an element. It modifies the coordinate space of the CSS visual formatting model.
  • z-index: The z-index CSS property sets the z-order of a positioned element and its descendants or flex and grid items. Overlapping elements with a larger z-index cover those with a smaller one.
  • opacity: Makes whatever element or font transparent.
  • pointer-events: The pointer-events CSS property sets under what circumstances (if any) a particular graphic element can become the target of pointer events. Side Note: This is also good if your Font or element overlaps into a clickable area 
            
              .grid-header h1{
                font-size: clamp(3rem, 8vw, 7rem);
                text-transform: uppercase;
                letter-spacing: 0.08em;
                color: yellow;
            
                position: absolute;
                left:55%;
                top: 45%;
                transform: translate(-50%, -50%);
            
                text-shadow:
                    1px 2px 2px #ff66e9,
                    0 .3em .7em black;
                opacity: .80;
                z-index: 1;
                pointer-events: none;
            }

4. Pure CSS Hamburger Menu

CSS styling can do alot including making something like a hamburger menu

Hamburger menus can use Javascript or SCSS but the original CSS Hamburger Menu practically uses every CSS styling existing.

Hamburger Menu : MVP CSS

  • 1. cursor: The cursor CSS property sets the mouse cursor, if any, to show when the mouse pointer is over an element.
  • 2. Span styling to make a Hamburger
  • 3. transform: Animating the Hamburger to do things lick rotate onclick using transform

Creating a hamburger menu is very creative and in my opinion if you can create a hamburger menu with transition, you are CSS-ing good. For this the cursor can be used to change the point icon, but the traditional pointer to selct is used here. There is also a "not allowed", which I didn't know how to use it before. 

          
            #menuToggle input
          {
            display: block;
            width: 40px;
            height: 32px;
            position: absolute;
            top: -7px;
            left: -5px;
            
            cursor: pointer;
            
            opacity: 0; /* hide this */
            z-index: 2; /* and place it over the hamburger */
            
            -webkit-touch-callout: none;
          }

Using CSS to change the hamburger menu int a cross. Key styling is transform which is used to rotate span by 45 degrees, but turn the middle (nth child) to opacity:0 and the last one rotating again at 45 degrees in the opposite direction. In learning about Hamburger menus - I also learned that people take the time to make hamburger menus. They spend TIME. 

          
            #menuToggle input:checked ~ span
            {
              opacity: 1;
              transform: rotate(45deg) translate(-2px, -1px);
              background: #232323;
            }

            /*
            * But let's hide the middle one.
            */
            #menuToggle input:checked ~ span:nth-last-child(3)
            {
              opacity: 0;
              transform: rotate(0deg) scale(0.2, 0.2);
            }

            /*
            * Ohyeah and the last one should go the other direction
            */
            #menuToggle input:checked ~ span:nth-last-child(2)
            {
              transform: rotate(-45deg) translate(0, -1px);
            }

5. CSS - Animation

The transition CSS property is a shorthand property for transition-property, transition-duration, transition-timing-function, transition-delay, and transition-behavior. This is combined with embedding a gif.

I made this with a div with class, "animate-this", inside another div with the same class, to see what it would look like. I think it causes motion sickness.

          .animate-this{
            margin: auto;
            height: 400px;
            width: 400px;
            animation: 
                pulse 3s ease infinite alternate, 
                nudge 5s linear infinite alternate;
            border-radius: 3%;
            overflow: hidden;
            position:relative;
            }

            }

            }

                transform: translate(-150px, 0);
